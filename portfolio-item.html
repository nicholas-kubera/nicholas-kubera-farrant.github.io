Logstash to Logos

When one log collector is not enough...

The main objectives behind this project were the following:
• Setup logos in a docker container and ensure it ran correctly.
• Understand current logstash configuration and convert it to FluentD configuration.
• Make sure that all plugins work correctly in FluentD and are correctly correlated to the same fucnctionality as in Logstash.

What is Logos?

Logos is basically a wrapper around FluentD. Logos allows FluentD to parse logs and upload those Logs to the cloud (AWS). It has its own built in API as well.

Docker configuration

I had to play around with the docker container startup parameters eventually getting to a point where the final parameters below worked.

I had to create a docker volume for the configuration as a prerequisite and this can be seen in the startup argument configuration paths. Port configuration was needed for system / firewall logs as the container was running under docker user and any port < 1024 on Linux is protected. This solution worked as expected.
